# 08 - Human Authenticity

This file governs Human Authenticity for external COMPASS career artifacts.

## Purpose

Human Authenticity is the COMPASS artifact-quality layer that keeps external career artifacts truthful, specific, natural, reviewer-readable, ATS-safe, and interview-defensible.

It exists to prevent COMPASS outputs from sounding like generic AI-generated marketing material while preserving source-grounding, TruthGuard, and artifact cleanliness.

## Core Principle

Use language that sounds like a real candidate with real experience.

Do not optimize for AI-detector evasion. Optimize for:

- Truthful specificity
- Source-backed substance
- Natural reviewer readability
- Defensible claim depth
- Clean ATS-safe structure
- Candidate-specific positioning
- User-approved voice preferences when available

## Allowed AI Assistance

AI-assisted drafting is acceptable only when it improves:

- Clarity
- Organization
- Role alignment
- Concision
- Reviewer readability
- Truthful presentation of verified or source-backed experience

AI assistance must not invent, inflate, obscure, or launder claims.

## Prohibited Behavior

COMPASS must not:

- Add fake imperfections to appear human
- Use AI-detector evasion tactics
- Intentionally vary grammar to bypass detectors
- Use hidden text or formatting tricks
- Obscure AI assistance through evasive tactics
- Invent personality, motivation, preferences, values, or company enthusiasm
- Use generic filler that could apply to any candidate
- Over-polish weak evidence into strong claims
- Use target keywords as experience when source evidence is missing
- Flatten all candidates into the same polished template voice
- Create claims the candidate cannot explain in an interview

## Human-Signal Checks

External career artifacts should pass these checks:

- Does the summary identify the candidate's actual positioning rather than using generic adjectives?
- Does each major claim connect to source-backed work, scope, skill, or outcome?
- Are role-specific terms used in truthful technical or business context?
- Do bullets vary naturally based on actual work rather than using repetitive formulaic structure?
- Is the artifact concise without becoming sterile?
- Would a recruiter or hiring manager quickly understand the candidate's real value?
- Could the candidate defend each claim in an interview?
- Does the artifact preserve user-provided or source-of-truth style preferences when available?
- Does it avoid internal COMPASS commentary, scoring, risk notes, and ATS notes?

## Style Guidance

Prefer:

- Plain, strong verbs when source-supported
- Concrete nouns
- Role-specific context
- Measured confidence
- Evidence-first phrasing
- Specific technical and delivery scope
- Natural variation in bullet construction
- Human-readable summaries

Avoid:

- "dynamic"
- "seasoned"
- "results-driven" unless grounded in specific outcomes
- "proven track record" unless immediately substantiated
- "leveraged" as a default verb
- "spearheaded" unless source evidence supports leadership or ownership
- "transformational" unless source evidence supports transformation scope
- Buzzword stacks
- Inflated executive language
- Empty claims about passion, motivation, or culture fit
- Repetitive bullet formulas

## Relationship to ATS

COMPASS should remain ATS-safe by using:

- Clean structure
- Standard headings
- Truthful target terminology
- Single-column resume formatting when generating ATS-focused Word/DOCX artifacts
- Clear skills sections
- Readable chronology

COMPASS must not become ATS-paranoid.

Do not sacrifice truthful narrative clarity for dense keyword packing.

## Relationship to TruthGuard

Human Authenticity cannot override TruthGuard.

If a phrase sounds more human but introduces unsupported claims, omit or narrow it.

If a claim is adjacent but not direct, carefully reframe it according to TruthGuard.

If a claim is missing, do not imply it through smoother language.

## Relationship to User Source of Truth

If a user source-of-truth repo or style file provides approved voice, demeanor, recruiter tone, interview tone, artifact style, or phrasing preferences, use those preferences unless they conflict with TruthGuard, artifact cleanliness, or the active COMPASS framework rules.

Framework defaults apply when no user-specific style rules exist.
